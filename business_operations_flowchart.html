A. Executive Summary

Scrunch & Create operates on a hybrid e-commerce setup. The storefront is a modern React SPA, but actual checkout transitions to direct-to-owner WhatsApp conversation to close payments, finalize design customization, and establish logistics options.

1. Storefront App (On-Platform)

React 19 + Vite 7 SPA hosted on Vercel. Catalog browsing, filters, cart, and checkout forms are managed locally in browser localStorage mock APIs.

2. Checkout Handoff

Submitting the checkout form triggers WhatsApp deep-linking with pre-filled cart details, shipping address, and order subtotal directed to the boutique owner.

3. Logistics & Booking

Boutique owner manually validates payments via UPI, processes physical stock check, packages items, and books shipping labels in Delhivery/Shiprocket dashboards.

F. Exception & Error Flow

Visualizes the error recovery loops for payment failure, stock shortages, delivery delays, and Return to Origin (RTO).

graph TD
    StartException([Exception Occurred]) --> Type{Exception Type}

    %% Payment Exception
    Type -->|Payment Failed| NotifyFail[Notify Client on WhatsApp -> Ask to Retry UPI / Netbanking]
    
    %% Stock Exception
    Type -->|Out of Stock / Custom Delay| InformTime[Inform Buyer: Handmade creation requires 48 hrs -> Get Approval]

I. Bottlenecks & Optimization Recommendations

Target Area Current Manual Process Proposed Automation Path
Payment Collection Share UPI details -> check bank app -> confirm chat. Razorpay/UPI Gateway Add checkout redirect for automatic verification.
Shipment Booking Copy address from WhatsApp -> paste into Delhivery. Delhivery/Shiprocket API Create shipment records on order submission automatically.
Tracking Notifications Copy AWB link -> send manually in chat. WhatsApp Business API Send automated shipping status templates (AWB) to customers.
Inventory Count Catalog items always "In Stock" -> manual check in boxes. Admin Dashboard MongoDB-backed inventory editor and real-time checkout validations.
